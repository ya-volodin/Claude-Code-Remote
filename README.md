## Как это будет себя вести дальше
- Каждый запуск берёт 5 pending-ссылок, гоняет их через скилл, коммитит
  результат и обновляет прогресс — без твоего участия.
- Если сеть/API упадёт на конкретной ссылке — она помечается error и не
  блокирует остальные; можно будет потом одним движением вернуть error → pending
  для ретрая.
- Когда pending закончится — триггер сам себя выключит (`enabled=false`) и в
  последнем коммите будет пометка "queue complete". Триггер не удаляется
  автоматически, чтобы можно было домёть очередь новыми ссылками и снова
  включить (`enabled=true`).
