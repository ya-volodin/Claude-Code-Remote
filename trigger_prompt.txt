Ты продолжаешь автономную обработку очереди SEO-ревью для tabletka.ru.

Файлы репозитория:
- queue.json — список ссылок со статусами: pending | in_progress | done | error
- progress.json — счётчики и лог запусков
- output/ — куда складывать готовые .docx
- .claude/skills/seo-text-review/ — сам скилл (SKILL.md + scripts/)
- TRIGGER_ID.txt — id этого триггера (для самоотключения в конце)

Выполни за этот запуск:

1. Прочитай queue.json и progress.json.
2. Возьми первые batch_size (по умолчанию 5) элементов со статусом "pending".
   Если pending нет — сразу переходи к шагу 6 (завершение).
3. Для каждой ссылки из батча прогони скилл seo-text-review целиком:
   - целевой запрос определяй автоматически из H1 (не спрашивай пользователя);
   - регион 213 (Москва), топ-5, порог 85%;
   - при блокировке SERP API из-за сети — зафиксируй это как error с причиной
     "network: SERP API blocked", не выдумывай конкурентов;
   - готовый отчёт сохрани в output/{index}_<slug-от-url-или-h1>.docx, где {index} —
     поле "index" элемента из queue.json (порядковый номер позиции, с ведущими
     нулями до 3 знаков, например "017").
   Обрабатывай ссылки по очереди, не параллель. Если одна ссылка падает —
   зафиксируй error с текстом причины и переходи к следующей, не прерывай весь батч.
4. Обнови queue.json: для каждой обработанной ссылки проставь status
   (done/error), query, score, report_path.
5. Обнови progress.json: пересчитай total/done/error/pending, допиши в log
   запись {timestamp, url, status, score}, обнови last_run_at и last_run_batch.
6. Закоммить и запушь изменения (queue.json, progress.json, output/*.docx) в репозиторий
   с сообщением вида "seo-review batch: N done, M error, K pending".
7. Если после этого батча в queue.json больше нет status="pending" —
   прочитай TRIGGER_ID.txt и вызови update_trigger(trigger_id=<id>, enabled=false),
   чтобы триггер больше не срабатывал. В коммите отметь "queue complete".
8. Ничего не спрашивай у пользователя — если что-то неоднозначно (например,
   не получилось выделить H1), прими консервативное решение, зафиксируй это в
   логе как замечание и продолжай.
